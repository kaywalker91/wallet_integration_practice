## 1. Claude Code의 Flutter 개발 활용 방식

Claude Code는 단순한 코드 자동완성을 넘어 자율적(agentic) 코딩을 지원하는 도구로, 개발자가 복잡한 소프트웨어 프로젝트를 계획하고, 작성하며, 디버깅하고, 배포하는 전 과정을 자동화하거나 간소화합니다.

**주요 활용 분야:**

| 영역 | Claude Code 활용 | 구체적 기능 |
|------|-----------------|------------|
| **코드 생성** | 보일러플레이트, 위젯, 상태관리 코드 자동 생성 | Riverpod, Flutter Hooks 패턴 구현 |
| **자동화** | 반복 작업 자동화 (플랜 업데이트, 커밋, 검사) | 커스텀 슬래시 명령어 |
| **테스트** | 유닛 테스트 작성 지원, 변경 후 자동 테스트 실행 | flutter_test, golden tests |
| **디버깅** | 네이티브 통합(권한, 음성인식) 문제 해결 | 오류 분석 및 수정 제안 |

## 2. Claude Code 기반 워크플로우 단계별 구성

### 🔄 표준 개발 플로우

```
┌─────────────────────────────────────────────────────────────┐
│  1단계: 프로젝트 초기화                                        │
│  ├── /init 실행 → CLAUDE.md 생성                             │
│  └── 프로젝트 개요, 아키텍처, 코딩 가이드라인 자동 분석            │
├─────────────────────────────────────────────────────────────┤
│  2단계: 요구사항 정의                                          │
│  ├── initial-requirements.md 상세 작성                        │
│  └── 기능 요구, UI 세부사항, 비기능 요구(코드 스타일) 명시         │
├─────────────────────────────────────────────────────────────┤
│  3단계: Plan Mode 활용                                        │
│  ├── Shift+Tab 2회 → 플랜 모드 진입                           │
│  └── 요구사항 기반 PLAN.md 실행 계획 생성                       │
├─────────────────────────────────────────────────────────────┤
│  4단계: 단계별 구현 + 커밋                                     │
│  ├── 작업 완료 후 리뷰 + 커밋                                  │
│  └── 앱이 항상 테스트 가능 상태 유지                            │
├─────────────────────────────────────────────────────────────┤
│  5단계: 검증 및 배포                                          │
│  └── 테스트 실행, CI/CD 자동화                                 │
└─────────────────────────────────────────────────────────────┘
```

세부 요구사항을 작성하는 데 시간을 투자하고, Plan Mode를 사용해 구조화하며, 각 작업 후 변경사항을 리뷰하고 커밋하는 방식이 핵심입니다.

## 3. 기존 Flutter 개발 플로우와의 차이점

| 구분 | 기존 방식 | Claude Code 방식 |
|------|----------|-----------------|
| **프로젝트 이해** | 수동 코드 분석 | 디렉토리, 의존성, 파일 구조를 깊게 이해하고 복수 파일·모듈 수준의 리팩토링까지 가능 |
| **병렬 작업** | 불가능 | Sub-agents: UI, Logic, Backend, Testing 에이전트가 동시에 작업 |
| **작업 연속성** | IDE 의존 | Checkpoints: 진행 상황 저장, 문제 발생 시 즉시 롤백 |
| **문서화** | 별도 진행 | CLAUDE.md가 영구적 컨텍스트로 세션 간 유지 |
| **개발 시간** | 기준 | 반복 업무 자동화로 평균 개발 기간 40% 단축, 버그 수정 속도 50% 개선 |

## 4. 관련 도구 및 API 통합

### 🔌 MCP (Model Context Protocol) 통합

MCP는 2024년 11월 25일 Anthropic이 발표한 오픈소스 표준으로, AI 시스템과 데이터 소스 및 도구를 연결하는 범용 프로토콜입니다.

**Flutter 개발자를 위한 핵심 MCP 서버:**

| MCP 서버 | 기능 | 설치 방법 |
|---------|------|----------|
| **Flutter MCP** | 50,000+ 패키지의 실시간 문서 제공, 할루시네이션 방지 | `npx flutter-mcp` |
| **Dart/Flutter MCP** | 패키지 추가, 포맷터 실행, 분석기 쿼리, 스크린샷 캡처, 핫 리스타트 | dart.dev 공식 제공 |
| **DCM MCP** | 코드 분석/수정, 미사용 코드/파일 감지, 메트릭 계산 | DCM 1.31.0+ |
| **MCP Flutter Toolkit** | VM 서비스 연결, 스크린샷 캡처, 에러 모니터링, 앱 검사 | `mcp_toolkit` 패키지 |

**연결 아키텍처:**
```
Claude Code ←→ MCP Server ←→ Flutter Debug Service ←→ Your Flutter App
```

### 🔗 기타 통합

Figma 연동, Slack 연동, Jira 연동 등 외부 도구와의 MCP 통합이 가능하며, GitHub MCP 서버로 이슈, 브랜치, PR 관리, Fetch MCP로 flutter.dev 문서 참조도 지원됩니다.
